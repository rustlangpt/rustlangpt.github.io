Fundamentos da Linguagem Rust: Expressões, Statements e Funções.

Vamos distinguir: Expressões, Statements e Funções.

RustLangPT publicado em 27 September, 2025
3 min, 495 words

Tags: Fundamentos Iniciantes Funções

Rust é uma linguagem que se distingue pela sua abordagem única à programação de sistemas, combinando performance com segurança. Um dos aspectos fundamentais que todo o programador Rust deve dominar é a distinção entre expressões, statements e a definição de funções. Neste artigo, vamos explorar estes conceitos em profundidade, com exemplos práticos que demonstram como utilizá-los eficazmente.

Expressões vs Statements: A diferença fundamental

Em Rust, existe uma distinção clara entre expressões e statements, conceito que é fundamental para compreender como a linguagem funciona.

Expressões

Uma expressão é qualquer código que avalia para obter um valor. Em Rust, quase tudo é uma expressão, incluindo blocos de código, chamadas de função, operações matemáticas, e muito mais.

// Expressões simples
let x = 5;          // 5 é uma expressão
let y = x + 1;      // x + 1  em si é uma expressão
let z = {           // Este bloco é uma expressão
    let a = 3;
    a + 1           // Note: sem ponto e vírgula!
};                  // z = 4

// Expressões condicionais
let numero = if true { 42 } else { 0 };

// Expressões de match
let valor = match x {
    1 => "um",
    2 => "dois",
    _ => "outro"
};

Ponto crucial: Expressões não terminam com ponto e vírgula. Se adicionarmos um ponto e vírgula, transformamos a expressão num statement.

Statements

Um statement é uma instrução que executa uma ação mas não retorna um valor. Em Rust, existem dois tipos principais de statements:

  1. Declaration statements - declarações de variáveis e funções
  2. Expression statements - expressões seguidas de ponto e vírgula
// Declaration statements
let x = 5;                    // Declaração de variável x e a respectiva atribuição
fn minha_funcao() {}          // Declaração de função 

// Expression statements
x + 1;                        // Expressão + ponto e vírgula = statement
println!("Hello, world!");    // Macro call statement

Exemplo Prático: Blocos de Código

Os blocos de código em Rust são expressões quando não terminam com ponto e vírgula:

fn main() {
    let resultado = {
        let a = 10;
        let b = 20;
        a + b        // Expressão: retorna 30
    };
    println!("Resultado: {}", resultado); // Output: Resultado: 30

    let sem_retorno = {
        let a = 10;
        let b = 20;
        a + b;       // Statement: não retorna valor
    };
    println!("Sem retorno: {:?}", sem_retorno); // Output: Sem retorno: ()
}

Funções em Rust: Definição e Estrutura

As funções em Rust seguem uma sintaxe específica e têm características únicas que as tornam poderosas e seguras.

Sintaxe Básica das Funções

fn nome_da_funcao(parametro1: tipo1, parametro2: tipo2) -> tipo_retorno {
    // corpo da função
    expressao_de_retorno
}

Exemplos de Funções

// Função simples sem parâmetros
fn saudacao() {
    println!("Olá, mundo!");
}

// Função com parâmetros
fn somar(a: i32, b: i32) -> i32 {
    a + b  // Expressão de retorno (sem ponto e vírgula)
}

// Função com múltiplos statements e uma expressão final
fn calcular_area_retangulo(largura: f64, altura: f64) -> f64 {
    println!("Calculando área do retângulo...");  // Statement
    let area = largura * altura;                   // Statement
    area  // Expressão de retorno
}

// Função com return explícito
fn verificar_par(numero: i32) -> bool {
    if numero % 2 == 0 {
        return true;  // Return explícito
    }
    false  // Expressão de retorno implícita
}

Funções como Expressões

Em Rust, as chamadas de função são expressões e podem ser usadas em qualquer lugar onde uma expressão é esperada:

fn multiplicar(x: i32, y: i32) -> i32 {
    x * y
}

fn main() {
    // Usando função como expressão em diferentes contextos
    let resultado = multiplicar(5, 3);
    
    let condicional = if multiplicar(2, 3) > 5 {
        "maior"
    } else {
        "menor ou igual"
    };
    
    println!("Resultado: {}, Condição: {}", resultado, condicional);
}

Funções aninhadas

fn outer() -> i32 {
    // Função aninhada
    fn inner() -> i32 {
        42
    }

    inner() // Chamada da função aninhada
}

println!("Valor retornado: {}", outer());

Exemplos Avançados: Combinando conceitos.

Função que retorna uma Closure

fn criar_multiplicador(fator: i32) -> impl Fn(i32) -> i32 {  
    // Esta é uma higher-order function - uma função que retorna outra função.
    // impl Fn - retorna algo que implementa o trait Fn (uma closure)
    // (i32) -> i32 - a closure retornada aceita um i32 e retorna um i32

    move |x| x * fator  // Expressão que retorna uma closure 
    // move - palavra-chave crucial que força a closure a tomar ownership das variáveis capturadas
}

fn main() {
    let multiplicar_por_3 = criar_multiplicador(3);
    println!("5 * 3 = {}", multiplicar_por_3(5)); // Output: 5 * 3 = 15
}

Funções com Pattern Matching

enum Operacao {
    Soma(i32, i32),
    Multiplicacao(i32, i32),
    Divisao(i32, i32),
}

fn executar_operacao(op: Operacao) -> Result<i32, String> {
    match op {  // Match é uma expressão
        Operacao::Soma(a, b) => Ok(a + b),
        Operacao::Multiplicacao(a, b) => Ok(a * b),
        Operacao::Divisao(a, b) => {
            if b == 0 {
                Err("Divisão por zero não é permitida".to_string())
            } else {
                Ok(a / b)
            }
        }
    }
}

Função Recursiva

fn fibonacci(n: u32) -> u64 {
    match n {
        0 => 0,
        1 => 1,
        _ => fibonacci(n - 1) + fibonacci(n - 2)  // Expressão recursiva
    }
}

// Versão otimizada com iteração
fn fibonacci_iterativo(n: u32) -> u64 {
    if n <= 1 {
        return n as u64;
    }
    
    let mut a = 0;
    let mut b = 1;
    
    for _ in 2..=n {
        let temp = a + b;
        a = b;
        b = temp;
    }
    
    b  // Expressão de retorno
}

Características Especiais das Funções em Rust

Tipo Unit ()

Quando uma função não retorna um valor explícito, ela retorna o tipo unit ():

fn imprimir_mensagem(msg: &str) {
    println!("{}", msg);
    // Retorna implicitamente ()
}

// Equivalente a:
fn imprimir_mensagem_explicita(msg: &str) -> () {
    println!("{}", msg);
}

Funções com Lifetimes

fn maior_string<'a>(s1: &'a str, s2: &'a str) -> &'a str {
    if s1.len() > s2.len() {
        s1
    } else {
        s2
    }
}

fn main() {
    let string1 = "Rust";
    let string2 = "Programming";
    let resultado = maior_string(string1, string2);
    println!("Maior string: {}", resultado);
}

Funções Genéricas

fn comparar<T: PartialOrd>(a: T, b: T) -> T {
    if a > b {
        a
    } else {
        b
    }
}

fn main() {
    println!("Maior entre 5 e 10: {}", comparar(5, 10));
    println!("Maior entre 'a' e 'z': {}", comparar('a', 'z'));
    println!("Maior entre 3.14 e 2.71: {}", comparar(3.14, 2.71));
}

Boas Práticas e Padrões Comuns

1. Usar Expressões Quando Possível

// Evitar - desnecessariamente verboso
fn classificar_numero_ruim(n: i32) -> &'static str {
    if n > 0 {
        return "positivo";
    } else if n < 0 {
        return "negativo";
    } else {
        return "zero";
    }
}

// Preferir - mais idiomático
fn classificar_numero_bom(n: i32) -> &'static str {
    if n > 0 {
        "positivo"
    } else if n < 0 {
        "negativo"
    } else {
        "zero"
    }
}

2. Documentar Funções Complexas

/// Calcula o fatorial de um número usando iteração
/// 
/// # Argumentos
/// 
/// * `n` - Um inteiro não-negativo
/// 
/// # Retorno
/// 
/// Retorna o fatorial de `n` ou `None` se `n` for muito grande
/// 
/// # Exemplos
/// 
/// ```
/// assert_eq!(fatorial(5), Some(120));
/// assert_eq!(fatorial(0), Some(1));
/// ```
fn fatorial(n: u32) -> Option<u64> {
    match n {
        0 => Some(1),
        1..=20 => {
            let mut resultado = 1;
            for i in 2..=n {
                resultado *= i as u64;
            }
            Some(resultado)
        }
        _ => None,  // Número muito grande
    }
}

3. Usar Pattern Matching Efetivamente

fn processar_resultado<T, E>(resultado: Result<T, E>) -> String 
where 
    T: std::fmt::Display,
    E: std::fmt::Display,
{
    match resultado {
        Ok(valor) => format!("Sucesso: {}", valor),
        Err(erro) => format!("Erro: {}", erro),
    }
}

Conclusão

A compreensão da diferença entre expressões e statements, juntamente com o domínio da definição de funções, é fundamental para escrever código Rust idiomático e eficiente. As expressões proporcionam uma forma concisa de retornar valores, enquanto os statements executam ações. As funções em Rust, por sua vez, oferecem um sistema robusto e flexível para organizar código, com suporte a generics, lifetimes e pattern matching.

Estes conceitos são a base sobre a qual se constrói o conhecimento mais avançado em Rust, incluindo ownership, borrowing e o sistema de tipos. Dominar estes fundamentos permitirá que escrevas código mais expressivo, seguro e performante.
